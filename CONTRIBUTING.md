# 🤝 Contributing to HESOYAM

First off — **thank you**. Every contribution makes this arsenal sharper for the entire Claude Code community.

## 🎯 Contribution Types

### 1. Playbooks (Most Wanted)

Real orchestration workflows you've tested in production. Place them in `playbooks/`.

**Format:**

```markdown
# Playbook: [Name]

## When to Use
[Scenario description]

## Prerequisites
[What needs to be installed]

## Steps
1. [Step with specific commands]
2. ...

## Expected Outcome
[What success looks like]

## Tips & Gotchas
[Hard-won knowledge]
```

### 2. Stack Configs

ECC/oh-my-claudecode configs tuned for specific tech stacks. Place them in `stacks/your-stack/`.

Include: CLAUDE.md, relevant skills, agent configs, recommended MCPs, and a README explaining the choices.

### 3. Knowledge Pipelines

Novel ways to connect tools in the knowledge layer. Place them in `knowledge/`.

### 4. Recipes

Step-by-step guides for specific outcomes. Place them in `community/recipes/`.

### 5. Showcase

Screenshots, recordings, or writeups of your setup. Place them in `community/showcase/`.

### 6. Translations

Translate any guide into another language. Place translated files alongside originals with a language suffix (e.g., `token-optimization.hi.md` for Hindi).

---

## 📋 PR Process

1. **Fork** this repo
2. **Create a branch** from `main` with a descriptive name
3. **Make your changes** following the structure and format guidelines
4. **Test your contribution** — if it's a config, make sure it actually works
5. **Open a PR** with:
   - Clear title describing the contribution
   - Brief explanation of what and why
   - Any relevant screenshots or recordings
6. **Respond to feedback** — we review PRs within 48 hours

## 🚫 What We Don't Accept

- Promotional content or spam
- Configs that haven't been tested
- Content that duplicates what's already in the referenced repos (link instead)
- Anything that requires paid services without a free alternative
